# תהליך עבודה סטנדרטי - פיתוח מערכת בדיקת תלושי שכר

## שלב 1 - אתחול הפרויקט (1-7/6/2025)
1. **הגדרת מטרות ויעדים**
   - פגישה עם צוות משאבי אנוש פנדה טק
   - תיעוד כללי עסק וחריגים קיימים
   - הגדרת KPIs למדידת הצלחה
   
2. **מינוי צוות הפרויקט**
   - ליתאי - מנהלת ואנליסטית
   - חשבת שכר - יועצת מקצועית
   - Claude AI - פיתוח טכני
   
3. **הכנת סביבת עבודה**
   - יצירת repository ב-GitHub
   - הגדרת תיקיית פרויקט מקומית
   - התקנת כלי פיתוח נדרשים

## שלב 2 - אפיון ותכנון (8-14/6/2025)
1. **אפיון פונקציונלי**
   - מיפוי תהליך בדיקת תלושים נוכחי
   - הגדרת 5 בדיקות אוטומטיות
   - עיצוב חוויית משתמש (UX)
   
2. **עיצוב טכני**
   - ארכיטקטורת client-side בלבד
   - בחירת ספריות: XLSX.js, Chart.js
   - תכנון מבנה נתונים
   
3. **עיצוב ממשק**
   - Wireframes ל-4 הטאבים
   - התאמה לעיצוב המותג
   - נגישות ורספונסיביות

## שלב 3 - פיתוח (15-25/6/2025)
1. **Sprint 1 - תשתית בסיסית**
   - מבנה HTML וסגנונות CSS
   - העלאת וקריאת קבצים
   - ניתוח נתוני Excel
   
2. **Sprint 2 - מנוע בדיקות**
   - פיתוח 5 הבדיקות האוטומטיות
   - לוגיקת זיהוי חריגות
   - חישוב חיסכון פוטנציאלי
   
3. **Sprint 3 - ממשק משתמש**
   - דשבורד דינמי
   - כרטיסי עובדים אינטראקטיביים

## שלב 4 - בדיקות מקיפות (26/6-5/7/2025)

   - בדיקת כל פונקציה בנפרד
   - תרחישי קצה
   
2. **בדיקות אינטגרציה**

   - תיקוני באגים מיידיים
